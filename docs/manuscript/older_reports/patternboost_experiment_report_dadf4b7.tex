\begin{abstract}
This report summarizes the current state of the PatternBoost geometric
extremal-search experiments.  The active scope is restricted to three
problems: maximum independent set of rectangles (MISR) LP-gap search,
unit-square line-stabbing LP-gap search, and recursive guillotine hardness.
The report also records a separate exploratory appendix run for
\code{epsilon\_net} and \code{graph\_separation}.  The current best audited
main-track values are $1.4$ for MISR, $1.5000000000000004$ for unit-square
stabbing, and $0.3$ for guillotine hardness.  The exploratory run reached
$1.4545454545454546$ for \code{epsilon\_net} and no exact positive
\code{graph\_separation} witness.  These values should be interpreted as the
present state of a structured computational search, not as final
publication-quality theorems.  All reported scores must be recomputed by exact
verifiers from standalone certificates before they are used in a manuscript.
\end{abstract}

\section{Scope and evidence boundaries}

The main paper track currently studies three extremal-search problems:
\begin{enumerate}
\item \textbf{MISR LP gap:} find axis-parallel rectangle families with large
LP relaxation gap for maximum independent set of rectangles.
\item \textbf{Unit-square stabbing:} find unit-square families with large
integer/LP gap for stabbing by horizontal and vertical lines.
\item \textbf{Guillotine hardness:} find disjoint rectangle packings for which
every recursive guillotine strategy destroys a large fraction of rectangles.
\end{enumerate}

Two additional tasks, \code{epsilon\_net} and \code{graph\_separation}, are
implemented as exploratory appendix tasks and should not be mixed with the main
three-problem table.  All evidence from the discarded
\code{square-stabbing-14-9} package is excluded.  In particular, the old
$14/9$ and $20/13$ square-stabbing values should not be quoted as current
evidence unless they are reimplemented and audited inside the allowed code
path.

The main scientific lesson is that generic coordinate mutation is not enough.
The useful progress comes when the representation and surrogate are shaped
like the certificate being sought:
\begin{enumerate}
\item MISR improves when the search directly optimizes exact LP-gap pressure
and uses sparse or triangle-free conflict structure as a prior.
\item Unit-square stabbing improves when the search works with line-square
incidence and exact set-cover scoring, not only raw square coordinates.
\item Guillotine improves when the search targets recursive nonseparability
and exact saved-set witnesses, not only first-cut obstruction.
\end{enumerate}

The current values are real but modest.  More wall time and more epochs alone
are unlikely to cross the main barriers unless the construction families and
verifier feedback loops are improved.

\section{Problem 1: MISR LP-gap search}

\subsection{Objective and exact verifier}

For a finite family $\Rcal$ of axis-parallel rectangles, the score is
\[
        \score_{\rm MISR}(\Rcal)=
        \frac{\alpha^*(\Rcal)}{\alpha(\Rcal)}.
\]
Here $\alpha(\Rcal)$ is the maximum number of pairwise-disjoint rectangles and
$\alpha^*(\Rcal)$ is the standard clique LP optimum.  The exact verifier must
construct the intersection graph, enumerate the clique or point constraints
needed for the LP, solve the LP and integral problem to optimality, and save
the objective components.

\subsection{Current best result}

The best current-code warm-start row has score $1.4$ with:
\[
\code{triangle\_free\_rect}
\quad
\code{lp\_dual\_pivot}
\quad
\code{exact\_lp\_gap\_pressure}.
\]

The archived visual certificate shown in Figure~\ref{fig:misr-best} records a
related old-framework MISR best with exact components:
\begin{itemize}
\item $n=34$ rectangles,
\item $\alpha^*=4.200000000000002$,
\item $\alpha=3$,
\item $\alpha^*/\alpha=1.4000000000000006$,
\item certificate hash stored in
\code{docs/assets/study\_assets/misr\_old\_best\_1p4.json}.
\end{itemize}

\subsection{Interpretation}

The gap depends on two quantities at once: the LP value must remain high while
the exact independent-set optimum remains small.  The current best examples
live in a small-denominator regime with $\alpha$ around 3, where every
fractional LP improvement is amplified.

The search has not yet shown a robust way to scale the $1.4$ phenomenon into a
larger family.  It can recover finite certificates and nearby variants, but it
does not yet reliably compose them while preserving the LP/integral imbalance.

The imported triangle-free local/program track has a lower exact score
$1.1851851851851851$ on 64 rectangles, but it is structurally important because
it enforces a sparse-conflict condition.  The best use of that track is not to
replace the unrestricted record search, but to split MISR into two explicit
tracks: unrestricted LP-gap records and triangle-free or sparse-conflict
records.

\subsection{Recommended next MISR changes}

\begin{enumerate}
\item Keep separate tables for unrestricted MISR and triangle-free/sparse MISR.
\item Add exact LP-dual features to triangle-free mutation; triangle-free
feasibility alone is not enough.
\item Mine motifs from the $1.4$ certificate and use them as optional mutation
templates, while keeping neutral matrix rows randomly generated.
\item Add a composition audit: after duplicating or lifting a motif, verify
whether $\alpha^*$ grows faster than $\alpha$.
\item Recover and audit any historical $1.5$ MISR example before it is cited.
\end{enumerate}

\subsection{Interpretation}

Unit-square stabbing is essentially a set-cover integrality-gap problem with a
geometric realization constraint.  Raw coordinate mutation is weak because most
coordinate moves do not preserve the incidence obstruction.  The better code
paths work because they search and score in terms of the critical
line-square incidence matrix.

The current value $1.5$ is meaningful, but it is also a plateau.  The search is
rediscovering compact $3/2$ behavior.  To beat it, the allowed code path needs
structured generators: staircase, ladder, cyclic threshold, or other
incidence-preserving families.  The discarded package should remain excluded,
but its mathematical lesson is clear: stronger gaps are unlikely to appear
from isolated local coordinate moves.

\subsection{Recommended next unit-square changes}

\begin{enumerate}
\item Keep \code{square-stabbing-14-9} excluded.
\item Reimplement only legitimate mathematical ideas inside the allowed code
path: staircase-like families, ladder families, cyclic threshold patterns, and
incidence-preserving generators.
\item Add richer certificate export: square coordinates, critical lines,
integer stabbing set, LP primal/dual witnesses, hash, and rendering.
\item Add mutations that preserve high-value incidence structure.  For example,
move a square only when it remains incident to multiple active critical lines.
\item Treat $1.5$ as the immediate barrier.  The next goal is a verified value
strictly above $3/2$ inside the allowed code path.
\end{enumerate}

\section{Problem 3: recursive guillotine hardness}

\subsection{Objective and exact verifier}

For a pairwise-disjoint family $\Rcal$ of axis-parallel rectangles inside a
bounding box, the score is
\[
        \score_{\rm guill}(\Rcal)
        =
        1-\frac{\save(\Rcal)}{|\Rcal|}.
\]
Here $\save(\Rcal)$ is the maximum number of rectangles that can be preserved
by an optimal recursive guillotine-cut strategy.  A cut destroys every
rectangle it crosses and then recurses on the rectangles strictly on each side.

The exact verifier is a recursive dynamic program over coordinate-induced
subwindows.  It must save the DP value, the chosen cut tree, the number of DP
states, the saved count, the destroyed count, and the exact destroyed fraction.

\subsection{Current best result}

The best current score is
\[
        0.3.
\]
The warm-start row reached this value with:
\[
\code{rect\_direct\_disjoint / packing\_resize / k\_subset\_nonseparability}.
\]
That row had 10 rectangles and destroyed 3.

The exact $n=40$ audit artifact gives the same destroyed fraction at larger
scale:
\begin{itemize}
\item $n=40$,
\item exact maximum saved: 28,
\item exact minimum destroyed: 12,
\item exact destroyed fraction: $12/40=0.30000000000000004$,
\item DP states: 2478,
\item target half-destruction condition: not met.
\end{itemize}

\subsection{Interpretation}

Guillotine is the most deceptive of the three problems.  A construction can
block many first cuts and still be easy after recursion.  Likewise, many
random large subsets can be nonseparable while exact DP still finds a large
saved subset.

The $n=40$ artifact is the clearest example.  The sampled obstruction signal
was strong, but the exact audit found \(\save=28\).  Therefore the exact score
was only $12/40=0.3$.  The missing ingredient is not just larger layouts.  The
search must use the exact saved set and cut tree as feedback, then add blockers
against the actual recursive escape routes.

\subsection{Recommended next guillotine changes}

\begin{enumerate}
\item Extract the exact saved set and cut tree from every high-scoring
candidate.
\item Convert saved-set witnesses into mutation targets: add blockers across
the exact split lines and move rectangles that enable easy recursive
separation.
\item Penalize candidates by attack-found or exact \(\save(\Rcal)\), not only
by sampled nonseparable subset fraction.
\item Build recursively interlocked cores so that every large subwindow
inherits a hard obstruction.
\item Use exact DP earlier on medium-size candidates to avoid optimizing a
misleading first-cut or sampled-subset proxy for hours.
\item Treat $0.33+$ as the next practical target before attempting a
half-destruction result.
\end{enumerate}

\section{Why the current values are stuck}

The bottleneck is not simply the number of epochs.  PatternBoost can train a
model over the current elite distribution, but the model cannot repair a
misaligned representation or a misleading surrogate.  The current plateaus are:

\begin{table}[htbp]
\centering
\caption{Current plateaus and likely causes.}
\begin{tabularx}{\textwidth}{p{0.19\textwidth}p{0.16\textwidth}X}
\toprule
Problem & Plateau & Likely cause \\
\midrule
MISR & $1.4$ & The search can recover finite certificates but does not yet
compose them while preserving the LP/integral imbalance. \\
Unit-square & $1.5$ & The search repeatedly finds compact $3/2$ set-cover
behavior; stronger allowed structured generators are not yet implemented. \\
Guillotine & $0.3$ & Local or sampled hardness overestimates exact recursive
hardness, and exact saved-set witnesses are not yet driving mutation. \\
\bottomrule
\end{tabularx}
\end{table}

The next meaningful improvements are likely to come from construction-family
engineering and verifier feedback, not from simply increasing the number of
random runs.

\section{Audit rules for reporting}

Before any value is quoted in a paper table, all of the following should hold:
\begin{enumerate}
\item The candidate appears in a completed \code{summary.json} row, not only in
a checkpoint.
\item The row has \code{return\_code = 0} or a clean and explained termination
status.
\item The stop reason is recorded.
\item The certificate path exists.
\item \code{multilevel verify} or \code{multilevel audit} recomputes the same
value.
\item The objective components are saved: \(\alpha^*,\alpha\) for MISR;
\(\tau,\tau^*\) for unit-square stabbing; \(n,\save,n-\save\) for guillotine.
\item Wall-time-killed rows are used only for debugging unless resumed and
audited.
\item Figures are treated as illustrations, not proof objects.
\item Excluded packages remain excluded, and exploratory tasks remain separate
from the main three-problem table.
\end{enumerate}
